# SOLID принципы:

Single Responsibility Principle
- `BankAccount` - отвечает только за данные счета
- `Category` - отвечает только за данные категории
- `Operation` - отвечает только за данные операции
- `BankAccountFacade` - управляет только операциями со счетами
- `OperationFacade` - управляет только операциями с транзакциями

Open-Closed Principle
- `IAnalyticsVisitor` и реализации `BalanceAnalytics`, `CategoryAnalytics` - система аналитики расширяема без изменения существующего кода
- `IExportVisitor` и реализации `JsonExportVisitor`, `CsvExportVisitor` - легко добавить новые форматы экспорта

Liskov Substitution Principle
- Все реализации интерфейсов (`IStorage<T>`, `IAnalyticsVisitor`, `IExportVisitor`) взаимозаменяемы

Interface Segregation Principle
- Интерфейсы разделены по функциональности: `IBankAccountFacade`, `IOperationFacade`, `IBankFactory`

Dependency Inversion Principle
- Зависимости инжектируются через конструкторы
- Используются абстракции (`IStorage<T>`, `IBankFactory`) вместо конкретных реализаций

# GRASP принципы:

Information Expert
- `BankAccount` содержит логику изменения баланса и имени
- `Operation` содержит логику изменения описания

Creator
- `BankFactory` создает объекты доменной модели

Controller
- `FinancialAccountingApp` - выступает как контроллер, обрабатывающий пользовательские команды и координирующий работу системы
- `BankAccountFacade` - контроллер для операций со счетами
- `OperationFacade` - контроллер для операций с транзакциями

Low Coupling
- Классы слабо связаны через интерфейсы и Dependency Injection

High Cohesion
- Каждый класс имеет четко определенную зону ответственности

# Реализованные паттерны GoF

## 1. Фасад
Классы: `BankAccountFacade`, `OperationFacade`
Польза:Упрощает взаимодействие с сложной системой хранения данных, предоставляя простой интерфейс для основных операций со счетами и операциями.

## 2. Фабрика
Классы:`BankFactory`, `IBankFactory`
Польза: Избавляет класс от привязки к конкретным классам продуктов. Выделяет код производства продуктов в одно место, упрощая поддержку кода. Упрощает добавление новых продуктов в программу. Реализует принцип открытости/закрытости.

## 3. Прокси
Классы: `StorageProxy<T>`
Польза: Добавляет кэширование к хранилищу данных, улучшая производительность при повторных обращениях к одним и тем же данным.

## 4. Шаблонный метод
Классы:`DataImporterTemplate`, `JsonImporter`
Польза: Облегчает повторное использование кода.

## 5. Посетитель
Классы: `IAnalyticsVisitor`, `BalanceAnalytics`, `CategoryAnalytics`, `IExportVisitor`, `JsonExportVisitor`, `CsvExportVisitor`
Польза:   Упрощает добавление операций, работающих со сложными структурами объектов.Объединяет родственные операции в одном классе.

## 6. Стратегия 
Классы:Реализации интерфейсов `IStorage<T>`
Польза:Замена алгоритмов на лету. Изолирует код и данные алгоритмов от остальных классов. Уход от наследования к делегированию. Реализует принцип открытости/закрытости
